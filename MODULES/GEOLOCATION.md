# Геолокация / автоопределение региона

> [← Сводный план модулей](../MODULES.md) | Источник: [REQS.md](../REQS.md) §2 (критичные, #8) | [TZ.md](../TZ.md) §6.3

## Статус: 🔌🔧 Новый плагин + частичная доработка

**Рекомендуемые плагины:**
- **GeoTargetingWP** (платный) — геоконтент, редиректы
- **WooCommerce GeoIP** (встроен в WooCommerce, базовый)
- **IP2Location Country Blocker** (бесплатный)

## Основное требование

Автоматическое определение региона пользователя при открытии сайта. На основе региона: язык, валюта, доступность товаров, контент страниц.

## Покрытие

| Требование | Раздел ТЗ | Покрытие |
|------------|-----------|----------|
| Автоопределение языка по браузеру | 6.3 | 🔧 Polylang Pro — частично (по Accept-Language) |
| Автоопределение региона по IP | 6.3 | 🔌 Не покрыто — нужен GeoTargetingWP |
| Настройка параметров по регионам (цены, доступность) | 6.3 | 🔌🔧 GeoTargetingWP + кастомная логика |

## Что нужно сделать

1. **Установить GeoTargetingWP** (или аналог)
2. **Настроить автоопределение** региона по IP
3. **Связать с языком** (Polylang) и валютой (CURCY) — автопереключение
4. **Доработка:** кастомная логика для региональных цен / доступности товаров, если стандартных настроек плагина недостаточно

## План работ и оценка

1. Установка и базовая настройка геолокационного плагина — 2 ч
2. Настройка правил определения региона и fallback-сценариев — 3 ч
3. Интеграция с валютой, языком и контентными сценариями — 4 ч
4. Настройка редиректов / баннеров / подсказок по региону — 2 ч
5. Тестирование разных региональных сценариев — 1 ч

**Итого:** 12 ч
