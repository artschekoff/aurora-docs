# SMS-рассылка

> [← Сводный план модулей](../MODULES.md) | Источник: [REQS.md](../REQS.md) §2 (важные, #10) | [TZ.md](../TZ.md) §6.5, 6.8

## Статус: 🛠️ Кастомная разработка

**Нет готового плагина** для РФ-провайдеров с полной функциональностью.

## Основное требование

SMS-рассылка по аналогии с email: уведомления о статусах заказов, акции, персональные предложения, подтверждение смены телефона/пароля. Интеграция с российским SMS-провайдером.

| Требование | Раздел ТЗ | Покрытие |
|------------|-----------|----------|
| SMS-уведомления (статусы заказов, подтверждения) | 6.5 | 🛠️ Кастомная разработка |
| SMS-рассылка по расписанию (акции, предложения) | 6.5 | 🛠️ Кастомная разработка |
| Интеграция со службой SMS-рассылки | 6.8 | 🛠️ Кастомная разработка |

## Что нужно разработать

1. **Кастомный плагин** на базе REST API одного из провайдеров:
   - SMS.RU
   - SMSC.ru
   - Twilio (для международных отправок)
2. **Шаблоны SMS** — настраиваемые в админке
3. **Триггеры событий** — подтверждение телефона, смена статуса заказа, abandoned cart
4. **Рассылка по расписанию** — интеграция с WP Cron
5. **Настройки провайдера** — возможность смены из админки
6. **Логирование** — история отправки SMS

**Оценка сложности:** Средняя

## План работ и оценка

1. Выбор провайдера и проектирование SMS-контура — 4 ч
2. Разработка backend-интеграции с API провайдера — 8 ч
3. Настройка шаблонов, триггеров и расписаний — 6 ч
4. Реализация журнала отправок и админ-настроек — 4 ч
5. Тестирование сценариев и правки — 2 ч

**Итого:** 24 ч
